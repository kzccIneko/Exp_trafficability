# 数据叠加分析指南

## 📊 NC土壤湿度文件内容

### 文件信息
- **文件名**: `soil_moisture_2023.nc` (从 `era5_land_soil_moisture_2023.nc` 解压)
- **格式**: NetCDF (科学数据标准格式)
- **来源**: ERA5-Land 再分析数据

### 包含的变量

| 变量名 | 说明 | 单位 | 深度 |
|--------|------|------|------|
| `swvl1` | 表层土壤湿度 | m³/m³ | 0-7cm |
| `swvl2` | 浅层土壤湿度 | m³/m³ | 7-28cm |
| `swvl3` | 中层土壤湿度 | m³/m³ | 28-100cm |
| `swvl4` | 深层土壤湿度 | m³/m³ | 100-289cm |

### 空间范围
- **经度**: 100.9°E ~ 101.1°E (3个点)
- **纬度**: 29.9°N ~ 30.1°N (3个点)
- **分辨率**: 约0.1° (约9公里)

### 时间范围
- **时间**: 2023年1月-12月 (12个月)
- **维度**: `(12, 3, 3)` = (时间, 纬度, 经度)

## 📐 数据叠加方法

### 1️⃣ 土壤湿度叠加到DEM

**问题**: 土壤湿度分辨率(~9km)远小于DEM(12.5m)

**解决方案**: 空间插值

```python
import rasterio
import xarray as xr
import numpy as np
from scipy.interpolate import griddata

# 1. 加载DEM
with rasterio.open('dem_12m_resampled.tif') as src:
    dem = src.read(1)
    transform = src.transform
    bounds = src.bounds

# 2. 创建DEM坐标网格
rows, cols = dem.shape
lon = np.linspace(bounds.left, bounds.right, cols)
lat = np.linspace(bounds.top, bounds.bottom, rows)
lon_grid, lat_grid = np.meshgrid(lon, lat)

# 3. 加载土壤湿度
ds = xr.open_dataset('soil_moisture_2023.nc')
sm = ds['swvl1'].mean(dim='valid_time')

# 4. 获取土壤湿度坐标
sm_lon = ds.longitude.values
sm_lat = ds.latitude.values
sm_values = sm.values

# 5. 创建土壤湿度坐标网格
sm_lon_grid, sm_lat_grid = np.meshgrid(sm_lon, sm_lat)

# 6. 插值到DEM网格
points = np.column_stack([sm_lon_grid.flatten(), sm_lat_grid.flatten()])
values = sm_values.flatten()

sm_interpolated = griddata(
    points,
    values,
    (lon_grid, lat_grid),
    method='linear'  # 线性插值
)

# 现在 sm_interpolated 和 dem 具有相同的形状
print(f"DEM形状: {dem.shape}")
print(f"土壤湿度形状: {sm_interpolated.shape}")
```

### 2️⃣ 道路数据叠加到DEM

**方法**: 栅格化或缓冲区分析

```python
import geopandas as gpd
import rasterio
from rasterio.features import rasterize

# 1. 加载道路
roads = gpd.read_file('osm_roads.shp')

# 2. 加载DEM获取元数据
with rasterio.open('dem_12m_resampled.tif') as src:
    transform = src.transform
    crs = src.crs
    out_shape = (src.height, src.width)

# 3. 栅格化道路
# 创建道路类型字典
road_types = {
    'major': ['trunk', 'primary', 'secondary'],
    'minor': ['tertiary', 'residential'],
    'track': ['track', 'service'],
    'path': ['footway', 'path']
}

# 为每种道路类型创建栅格
for road_type, highway_values in road_types.items():
    # 筛选道路
    selected_roads = roads[roads['highway'].isin(highway_values)]

    # 栅格化
    road_raster = rasterize(
        [(geom, 1) for geom in selected_roads.geometry],
        out_shape=out_shape,
        transform=transform,
        fill=0,
        dtype='uint8'
    )

    print(f"{road_type}: {road_raster.sum()} 个像素")
```

### 3️⃣ 水系数据叠加到DEM

**方法**: 计算到水系的距离

```python
import geopandas as gpd
import numpy as np
from scipy.ndimage import distance_transform_edt

# 1. 加载水系
waterways = gpd.read_file('osm_waterways.shp')

# 2. 创建水系栅格
water_raster = np.zeros(dem.shape)

for idx, water in waterways.iterrows():
    if water.geometry.geom_type == 'LineString':
        coords = np.array(water.geometry.coords)
        for coord in coords:
            # 转换为像素坐标
            col = int((coord[0] - bounds.left) / transform.a)
            row = int((coord[1] - bounds.top) / transform.e)

            if 0 <= row < dem.shape[0] and 0 <= col < dem.shape[1]:
                water_raster[row, col] = 1

# 3. 计算到水系的距离
water_distance = distance_transform_edt(1 - water_raster)
water_distance = water_distance * abs(transform.a)  # 转换为地理单位
```

### 4️⃣ 综合通行能力模型

**将所有因素叠加计算通行能力得分**

```python
def normalize(data):
    """归一化到0-1"""
    return (data - np.nanmin(data)) / (np.nanmax(data) - np.nanmin(data))

# 归一化各因素
slope_norm = normalize(slope)
sm_norm = normalize(sm_interpolated)
road_norm = normalize(road_density)
water_norm = normalize(water_distance)

# 权重分配
weights = {
    'slope': 0.4,           # 坡度影响最大
    'soil_moisture': 0.3,   # 土壤湿度
    'road_density': 0.2,    # 道路密度
    'water_distance': 0.1   # 到水系距离
}

# 计算通行能力得分（0-1，越高越容易通行）
trafficability = (
    weights['slope'] * (1 - slope_norm) +        # 坡度越小越好
    weights['soil_moisture'] * (1 - sm_norm) +    # 土壤越干越好
    weights['road_density'] * road_norm +          # 道路密度越高越好
    weights['water_distance'] * water_norm         # 距离水系越远越好
)
```

## 📊 各因素对通行能力的影响

| 因素 | 影响 | 权重 | 说明 |
|------|------|------|------|
| **坡度** | 最大 | 40% | 坡度越大，通行越困难 |
| **土壤湿度** | 大 | 30% | 土壤越湿，越容易打滑 |
| **道路密度** | 中 | 20% | 道路越多，通行越容易 |

## 🎯 分析结果解释

### 通行能力得分 (Trafficability Score)
- **0.0 - 0.3**: 通行困难（陡坡、湿地、无道路）
- **0.3 - 0.5**: 通行较困难（中等坡度、湿润土壤）
- **0.5 - 0.7**: 通行一般（平缓地形、干燥土壤）
- **0.7 - 1.0**: 通行容易（平坦、干燥、有道路）

## ⚠️ 注意事项

1. **坐标系统**: 所有数据必须使用相同的坐标系 (EPSG:4326)
2. **分辨率匹配**: 低分辨率数据需要插值到高分辨率
3. **NoData处理**: 处理无效值
4. **内存管理**: 大范围数据可能需要分块处理
