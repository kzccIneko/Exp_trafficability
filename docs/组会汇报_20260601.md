# 方向敏感越野通行能力建模与实验验证

**组会汇报 · 2026年6月1日**

---

## 一、研究背景与问题提出

### 1.1 研究目标

构建**越野环境下的通行代价图**，为无路网/少路网条件下的路径规划提供基础。

### 1.2 现有方法的问题

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **坡度标量化** | 传统方法用 `s = |∇z|` 作为代价，不区分方向 | 无法反映"上坡难、下坡易"的物理现实 |
| **忽略横坡风险** | 只考虑沿坡度方向的纵坡，忽略侧向坡度 | 沿等高线行驶时低估侧翻/侧滑风险 |
| **代价函数缺乏物理依据** | 随意选择线性/指数函数，无阈值设计依据 | 参数难以解释，结果难以验证 |

### 1.3 文老师提出的核心问题

> **"坡度事实上有可能是正向的推动作用，下坡呢？我们从来没考虑过下坡。"**

> **"我的坡度的阻碍作用是跟从A点到B点相关的...代价图是个可变的。"**

这两个问题指向同一个核心：**坡度对通行能力的影响必须考虑方向性**。

## 二、方法设计

### 2.1 核心创新：纵坡/横坡分解

**为什么需要分解？**

传统方法用标量坡度 `s = |∇z|`，但：
- 沿最大坡降上坡：纵坡最大，横坡为0
- 沿等高线行驶：纵坡为0，横坡最大（侧翻风险！）
- 斜向行驶：纵坡和横坡都有分量

**分解公式：**

设 DEM 梯度 `∇z = (p, q)`，行驶方向 `θ`：

```
纵坡: g_∥ = p·cos(θ) + q·sin(θ)    // 沿行驶方向的坡度分量
横坡: g_⊥ = -p·sin(θ) + q·cos(θ)   // 垂直行驶方向的坡度分量
```

**维度说明：** `p, q` 来自 `np.gradient(dem, cell_size)`，物理含义是 `tan(坡度角)`（无量纲），不是弧度。

### 2.2 纵坡阻碍度：三段式非对称函数

**为什么是三段式？**

回答老师的问题："下坡呢？我们从来没考虑过下坡。"

| 坡度区间 | 物理行为 | 阻碍度 |
|----------|----------|--------|
| **上坡** (g_∥ > 0) | 需要更大牵引力 | R_∥ = 1 - exp(-(g_∥/tan(α_u))²) |
| **缓下坡** (-tan(α_m) ≤ g_∥ ≤ 0) | 重力辅助，轻微下坡有利行驶 | R_∥ = 0（死区） |
| **陡下坡** (g_∥ < -tan(α_m)) | 需要更大制动力，存在失稳风险 | R_∥ = 1 - exp(-(\|g_∥\|-tan(α_m))²/tan(α_d)²) |

**为什么选择高斯衰减形式？**
1. **光滑性**：分段交界处无突变，避免梯度不连续
2. **有界性**：R ∈ [0, 1) 自然满足阻碍度范围约束
3. **物理直觉**：当 g = tan(α) 时，R = 1 - 1/e ≈ 0.632（"中等阻碍"的定义）
4. **快速饱和**：超过 α 后阻碍增长放缓，反映"已经很难走了，再陡也差不多"

**参数设计依据：**

| 参数 | 默认值 | 设计依据 |
|------|--------|----------|
| α_u = 15° | 上坡阻碍尺度 | 普通越野车极限爬坡角30°~40°，15°时已明显感到困难 |
| α_m = 5° | 缓下坡容许角 | 5°以内下坡对大多数车辆几乎无感（类似高速公路最大纵坡） |
| α_d = 15° | 陡下坡风险尺度 | 与α_u对称，超过5°后阻碍增长速率与上坡相同 |

### 2.3 横坡阻碍度

**为什么横坡必须单独建模？**

> 横坡影响的是车辆的**侧向稳定性**，与纵坡的牵引/制动需求是完全不同的物理机制。

```
R_⊥ = 1 - exp(-(|g_⊥| / tan(α_r))²)
```

**为什么 α_r = 10° < α_u = 15°？**

车辆对横坡更敏感——10°横坡已让人明显感到倾斜，而10°纵坡上坡几乎无感。文献中侧翻临界横坡角通常在25°~35°，10°作为"中等阻碍"的尺度参数是保守但安全的选择。

### 2.4 非补偿合成

**为什么不用加权平均？**

如果用 `R = w·R_∥ + (1-w)·R_⊥`，当 R_∥=0.9、R_⊥=0.1 时，R=0.5（中等阻碍）——不合理，因为纵坡已经很难走了。

**非补偿合成公式：**
```
R_slope = 1 - (1 - R_∥)(1 - R_⊥)
```

**数学性质：**
- R ≥ max(R_∥, R_⊥)：综合阻碍不低于任一分量
- 物理解读：通过能力 = 纵向通过率 × 横向通过率

### 2.5 边代价转换

**为什么用 -ln(P) 而非直接用 R？**

路径规划需要**可累加的代价**。`-ln(P)` 将通过能力转换为可累加的代价：

```
c(x,y,θ) = -ln(1 - R + ε)
```

| R | P = 1-R | c = -ln(P) | 含义 |
|---|---------|------------|------|
| 0 | 1 | 0 | 完全畅通 |
| 0.5 | 0.5 | 0.693 | 中等阻碍 |
| 0.9 | 0.1 | 2.303 | 高阻碍 |
| →1 | →0 | →∞ | 不可通行 |

## 三、实验设计与验证

### 3.1 实验设置

| 参数 | 值 | 说明 |
|------|-----|------|
| DEM尺寸 | 80×80 | 合成DEM，含山脊/山谷/随机起伏 |
| 栅格大小 | 30m | SRTM 30m等效分辨率 |
| 坡度范围 | 0°~35.2° | 覆盖缓坡到陡坡 |
| 方向数 | 8 | 8邻域（间隔45°） |

**三模型定义：**

| 模型 | 类型 | 代价计算 | 特点 |
|------|------|----------|------|
| **B0** | 标量 | C = f(\|∇z\|)，不区分方向 | 传统方法基线 |
| **B1** | 方向（仅纵坡） | C = f(g_∥)，忽略横坡 | 方向敏感但不完整 |
| **Ours** | 方向（纵坡+横坡） | C = f(g_∥, g_⊥)，非补偿合成 | 完整方向敏感模型 |

### 3.2 实验一：三模型代价场对比

| 指标 | B0（标量） | B1（仅纵坡） | Ours（纵坡+横坡） |
|------|------------|--------------|-------------------|
| 代价范围 | [0.0000, 2.0895] | [0.0000, 2.0895] | [0.0000, 2.0895] |
| 平均代价 | 0.4968 | 0.3645 | 0.7762 |
| AI范围 | — | [0.0000, 1.0000] | [0.0000, 1.0000] |
| 平均AI | — | 0.9999 | 0.7637 |

**关键发现：**
- B1的AI≈1.0（极高各向异性），因为纵坡函数的"死区"导致某一方向代价为0而其他方向非0
- Ours的AI≈0.76（中等各向异性），因为横坡分量在所有方向都贡献了基础代价
- Ours的平均代价高于B0和B1，反映了横坡的额外贡献——**这是合理的，不是bug**

### 3.3 实验二：路径规划对比

| 指标 | B0 | B1 | Ours |
|------|-----|-----|------|
| 路径步数 | 91 | 120 | 93 |
| 平均代价/步 | 0.4593 | 0.0011 | 0.5538 |
| 总代价 | 41.7939 | 0.1279 | 51.5040 |

**路径特征分析：**
- B1平均代价极低（0.0011），因为死区导致轻微下坡方向代价为0，路径几乎只走缓下坡方向
- Ours路径步数（93步）与B0（91步）接近，但平均代价更高，因为横坡增加了额外代价
- Ours路径会**主动避开横坡大的区域**（如沿山坡等高线），这是B0和B1做不到的

### 3.4 实验三：地形流线验证

**验证方法说明：**

> 传统验证用OSM道路数据，但存在循环论证风险（用模型自己的路径模拟道路）。
> 本实验采用**地形梯度下降流线**作为验证基准：流线方向 = 梯度方向（aspect），这是物理上的最速下降方向，不依赖任何模型。

| 指标 | B0 | B1 | Ours | 说明 |
|------|-----|-----|------|------|
| **AUC** | 0.5343 | 0.2764 | **0.6159** | >0.5 = 能区分 |
| **DC** | — | 0.7193 | **0.9852** | 越高 = 方向越一致 |
| **ΔC 均值** | — | — | 0.2750 | >0 = 流线代价更低 |
| **Wilcoxon p** | — | — | 4.55e-74 | <<0.05 = 显著 |

**结果解读：**
- **AUC排序**: Ours (0.616) > B0 (0.534) > B1 (0.276)
- Ours > 0.5：模型能区分流线/非流线点（流线点的综合代价更低）
- B1 < 0.5：因为流线沿梯度上坡（最大纵坡），B1只看纵坡→流线代价反而最高
- Ours > B0的原因：流线方向 g_⊥ = 0，Ours模型正确识别了"无横坡"的优势
- DC = 0.98：Ours的最优方向与流线方向高度一致（cos差异≈0）
- Wilcoxon p << 0.05：配对代价差显著大于0，统计上确认流线代价更低

### 3.5 实验四：参数敏感性分析

**α_u（上坡阻碍尺度角）：**

| α_u (°) | 平均代价 | 平均AI | 物理解读 |
|----------|----------|--------|----------|
| 5 | 2.3978 | 0.7527 | 极敏感：5°就达63%阻碍，代价极高 |
| 10 | 1.1248 | 0.7577 | 较敏感 |
| **15** | **0.7762** | **0.7637** | **默认值：平衡点** |
| 20 | 0.6078 | 0.7674 | 较宽松 |
| 30 | 0.4368 | 0.7720 | 极宽松：30°才达63%阻碍 |

**α_r（横坡风险尺度角）——影响最大的参数：**

| α_r (°) | 平均代价 | 平均AI | 物理解读 |
|----------|----------|--------|----------|
| 5 | 2.5376 | 0.8793 | 极敏感：5°横坡就达63%阻碍，代价暴涨 |
| 8 | 1.2829 | 0.8156 | 较敏感 |
| **10** | **0.7762** | **0.7637** | **默认值：10°横坡达63%阻碍** |
| 15 | 0.4197 | 0.6920 | 宽松 |
| 20 | 0.3219 | 0.6545 | 极宽松：20°横坡才达63%阻碍 |

**设计依据：** α_r = 10°是保守但安全的选择——10°横坡已让人明显感到倾斜。

## 四、回答老师的问题

### 问题1：坡度不只是阻碍，还可能是助力

**回答：** 我们设计的三段式纵坡阻碍度函数已经解决了这个问题：
- **上坡** (g_∥ > 0)：阻碍作用，R_∥随坡度增大而增大
- **缓下坡** (-5° ≤ g_∥ ≤ 0)：**助力作用**，R_∥ = 0（死区），不增加代价
- **陡下坡** (g_∥ < -5°)：阻碍作用重新增加（制动需求+失稳风险）

### 问题2：坡度的代价与行进方向相关

**回答：** 通过纵坡/横坡分解实现：
- 坡度和坡向是固定的（DEM决定）
- 唯一的变量是行进方向 θ
- 通过 `g_∥ = p·cos(θ) + q·sin(θ)` 计算沿行驶方向的坡度分量
- 代价场 `C(x,y,θ)` 是方向敏感的矢量场，不是标量场

### 问题3：方法太简单，缺乏理论深度

**回答：** 我们的方法有完整的理论支撑：
1. **纵坡/横坡分解**：基于向量投影的严格数学推导
2. **三段式非对称函数**：基于车辆物理行为的分段建模（牵引力/制动力/侧翻力矩）
3. **非补偿合成**：基于概率补集的数学性质（通过能力 = 纵向通过率 × 横向通过率）
4. **-ln(P)代价转换**：基于信息论的可累加代价设计
5. **参数设计依据**：基于文献中的车辆极限参数（爬坡角30°~40°、侧翻临界横坡角25°~35°）

### 问题4：缺乏定量验证

**回答：** 我们设计了三套定量验证指标：
1. **AUC**：区分流线/非流线点的能力（Ours: 0.616 > B0: 0.534 > B1: 0.276）
2. **DC**：最优方向与流线方向的一致性（Ours: 0.985）
3. **ΔC + Wilcoxon检验**：配对代价差的统计显著性（p << 0.05）

## 五、Bug修复记录

在实验过程中，我们发现并修复了6个关键Bug：

| Bug | 问题 | 修复方式 |
|-----|------|----------|
| #1 斜率维度不匹配 | `g_parallel`是tan值，但用`radians(α)`比较 | 所有阻碍度函数改为`tan(α)`尺度 |
| #2 边长重复计算 | `edge_cost`乘了`l_theta`，A*又乘`move_dist` | `edge_cost`只返回单位距离代价 |
| #3 负代价风险 | `ln`参数可能≤0 | 用`np.clip`保护 |
| #4 方向角周期性 | 0°和360°附近匹配错误 | 改用`min(\|diff\|, 2π-\|diff\|)` |
| #5 自验证OSM | 用模型自己的路径验证自己 | 改用梯度下降流线 |
| #6 边界路径效应 | 路径绕边界走 | 添加边界代价惩罚 |

## 七、总结

### 核心创新点

1. **方向敏感的矢量场**：将通行能力从标量场 `C(x,y)` 升级为方向敏感的矢量场 `C(x,y,θ)`
2. **纵坡/横坡分解**：基于向量投影，同时考虑牵引/制动需求和侧翻风险
3. **三段式非对称函数**：区分上坡（阻碍）、缓下坡（助力）、陡下坡（阻碍）三种物理行为
4. **非补偿合成**：确保任一分量阻碍很高时，综合阻碍也很高

### 实验验证结果

- Ours模型的AUC（0.616）优于B0（0.534）和B1（0.276）
- 方向一致性DC达到0.985，说明模型正确识别了最优方向
- 配对代价差统计显著（p << 0.05）

### 回答老师的核心问题

> **"坡度事实上有可能是正向的推动作用"**

是的，我们的三段式函数通过"死区"设计，将缓下坡（-5°~0°）的阻碍度设为0，实现了"下坡助力"的建模。

> **"坡度的代价与行进方向相关"**

是的，通过纵坡/横坡分解，代价场 `C(x,y,θ)` 成为方向敏感的矢量场，同一栅格在不同行驶方向上有不同的代价。
